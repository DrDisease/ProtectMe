Our Project



Context

Ever since the inception of social media networks, there have always been individuals interested in taking advantage of these platforms in order to spread malicous content and fullfill a given agenda, and in recent years, not only did these platforms' user base grow, but so did the number of people attempting these harmfull acts. However, although there have always been attempts to monitor and slow down the spread of this type of content, there has hardly ever been an effort to automatically analise all the published content in a self-contained manner, be it due to the lack of technology for that, be it through a lack of interest. The original goal of this project was to create a pipeline that would analyze social media content in this self-contained manner (analyze both text and other mediatic content and correlate them) with the help of current state of the art Natural Language Processing, Media Content Extraction and Machine Learning technologies, however, due to the lack of a data set that fullfills the requirements for this endeavour, we had to rethink our purpose.

Objectives and Purpose

After having our original purpose made obsolete, we decided to still make an effort to improve this area, now with a new and more realistic objective! The ProtectMe! project aims to provide the means to gather and automatically analyze content from a social media network, through the usage of Natural Language Processing(NLP) and Media Content Extraction Technologies in order to generate a dataset and the tools needed to analyze it. As such, our objectives for this project are:

  • Gather a list of functional and non-functional requirements
  • Analyze the current State of the Art for NLP and Media Content Extraction Technologies
  • Produce a content extraction pipeline
  • Automate the process of searching and analyzing social media content
  • Generate a data set with the already extracted and analyzed content
  • Produce a web interface to analyze the data set's contents as well as a simple interface to expand and export it

Users

This application only foresees one kind of user, a simple user who wants to checkif a social media post contains malicious content or not.From this original type of user, we can sub-categorize it into a registered orunregistered user. A unregistered user would be someone who wants to anonymously check a singlepost, as a single time event whereas a registered user would be someone whowould be using this service with some manner of frequency andwould like tokeep track of the post they searched and the results they got.

  • Common User
    •
    The common user is a user who uses the system in an anonymous manner, whodoesn’t desire to have a track record of his searches. As such,it’s only supposedto query a given social media post and analyze the results obtained.
  • Registered User
    •
    A registered user is a user who uses the system in some sort of regular manner,and has the need to keep a track record of his queries. As such, this user is ableto log into the application (as well as log out); check his search history and, likethe common user, query a social media post and analyze the query’s results.
